# Блог-платформа

Сучасна платформа для створення та читання статей, побудована з використанням Next.js та Tailwind CSS.

## 🚀 Опис проекту

Цей проект є повноцінною веб-платформою для блогінгу, яка дозволяє користувачам реєструватися, створювати власні публікації, редагувати їх та переглядати статті інших авторів. Платформа має сучасний мінімалістичний дизайн, підтримку темної теми та повністю адаптивний інтерфейс.

### Основні можливості:

- 🔐 **Автентифікація**: реєстрація, вхід та вихід з системи.
- 📝 **Керування статтями**: повний цикл CRUD (створення, читання, оновлення, видалення).
- 👤 **Панель керування**: особистий кабінет для авторів та адміністраторів.
- 📱 **Адаптивність**: коректне відображення на мобільних пристроях, планшетах та десктопах.
- 🎨 **Сучасний дизайн**: інтерфейс на базі Shadcn UI.

## 🛠 Технологічний стек

- **Framework**: [Next.js 15+](https://nextjs.org/) (App Router)
- **Styling**: [Tailwind CSS 4](https://tailwindcss.com/)
- **UI Components**: [Shadcn UI](https://ui.shadcn.com/) (Radix UI)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Forms**: [React Hook Form](https://react-hook-form.com/) + [Zod](https://zod.dev/)
- **State/Storage**: Client-side storage (localStorage) для симуляції бекенду

## ⚙️ Інструкція запуску

Для запуску проекту локально виконайте наступні кроки:

1. **Клонуйте репозиторій**:

   ```bash
   git clone <url-репозиторію>
   cd blog
   ```

2. **Встановіть залежності**:

   ```bash
   npm install
   # або
   pnpm install
   ```

3. **Запустіть сервер для розробки**:

   ```bash
   npm run dev
   ```

4. **Відкрийте проект**:
   Перейдіть за посиланням [http://localhost:3000](http://localhost:3000) у вашому браузері.

## 🏗 Архітектурні рішення

### 1. Симуляція бекенду (Client-side Persistence)

Для забезпечення працездатності проекту без зовнішньої бази даних було реалізовано шар сховища в `lib/storage.ts`, який використовує `localStorage`. Це дозволяє:

- Зберігати дані між оновленнями сторінки.
- Симулювати реальні HTTP-запити через затримки.
- Легко перенести логіку на реальний API у майбутньому.

### 2. Компонентний підхід та Shadcn UI

Використання Shadcn UI дозволило швидко створити доступні та стильні компоненти, які легко налаштовувати. Кожен компонент знаходиться у папці `components/ui` та є атомарною одиницею інтерфейсу.

### 3. Tailwind CSS 4

Проект використовує найновішу версію Tailwind CSS 4, що забезпечує швидку збірку стилів та нові можливості, такі як вбудовані змінні теми (inline theme) та покращену підтримку темного режиму без додаткових конфігураційних файлів.

### 4. Структура App Router

Використано структуру папок Next.js App Router для чіткого розділення сторінок, макетів (layouts) та серверних компонентів. Маршрути згруповані за функціональністю (наприклад, `/dashboard` для адміністративної частини).
